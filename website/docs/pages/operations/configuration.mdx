# Configuration [Every config option across every service]

Complete reference for all configuration options across FPC services.

## Configuration Hierarchy

All services follow the same precedence:

1. **YAML config file** — Base configuration (path from `CONFIG_PATH` env var)
2. **Environment variables** — Override specific values
3. **CLI arguments** — Override at runtime

Environment variables always take precedence over YAML values.

---

## Attestation Service

**Config file:** `services/attestation/config.example.yaml`

### Network

| YAML Key | Env Var | Default | Description |
|----------|---------|---------|-------------|
| `network_id` | `NETWORK_ID` | — | Aztec network identifier — **string**, e.g. `"aztec-testnet"` (not a numeric chain id) |
| `fpc_address` | `FPC_ADDRESS` | — | Deployed FPC contract address |
| `contract_variant` | — | `"fpc-v1"` | Contract flavour identifier emitted in discovery |
| `aztec_node_url` | `AZTEC_NODE_URL` | — | Aztec node PXE endpoint |
| `quote_base_url` | `QUOTE_BASE_URL` | derived from request headers | Public base URL emitted in `.well-known/fpc.json` — set this if behind a reverse proxy |

### Operator

| YAML Key | Env Var | Default | Description |
|----------|---------|---------|-------------|
| `operator_secret_provider` | — | `auto` | Key mode: `auto`, `env`, `config`, `kms`, `hsm` |
| `operator_address` | `OPERATOR_ADDRESS` | — | Operator L2 address (non-zero salt) |
| `operator_account_salt` | `OPERATOR_ACCOUNT_SALT` | — | Account salt (optional) |
| — | `OPERATOR_SECRET_KEY` | — | L2 operator Schnorr private key (plaintext) |
| — | `OPERATOR_SECRET_KEY_REF` | — | KMS / secret-manager reference for the key (**preferred in production**) |

:::tip[Prefer `_REF` variants in production]
Any `*_SECRET_KEY` env var has a matching `*_SECRET_KEY_REF` variant that resolves via the configured secret provider (KMS/HSM). Use the `_REF` form so plaintext keys never appear in process env or shell history.
:::

### Quote Rate Limiting

| YAML Key | Env Var | Default | Description |
|----------|---------|---------|-------------|
| — | `QUOTE_RATE_LIMIT_ENABLED` | `true` | Master toggle |
| — | `QUOTE_RATE_LIMIT_MAX_REQUESTS` | `60` | Max requests per window per key |
| — | `QUOTE_RATE_LIMIT_WINDOW_SECONDS` | `60` | Window length (seconds) |

### Assets

```yaml
supported_assets:
  - address: "0x..."
    name: "humanUSDC"           # Human-readable name surfaced in /accepted-assets
    market_rate_num: 1          # Base exchange rate numerator (asset units per 1 FeeJuice)
    market_rate_den: 1000       # Base exchange rate denominator
    fee_bips: 200               # Operator margin: 200 bips = 2%
```

Final rate computed as `rate_num = market_rate_num × (10000 + fee_bips)` / `rate_den = market_rate_den × 10000`. Per-asset `market_rate_num`, `market_rate_den`, and `fee_bips` overrides are all optional — omitted values inherit top-level defaults.

### Auto-Claim

Auto-claim completes the L2 side of the top-up bridge (claims the bridged Fee Juice into the FPC's balance). **Enabled by default** — set to `0` to disable.

| Env Var | Default | Description |
|---------|---------|-------------|
| `TOPUP_AUTOCLAIM_ENABLED` | `1` | `1` = enabled, `0` = disabled |
| `TOPUP_AUTOCLAIM_SECRET_KEY` | — | L2 claimer secret key (**required** in `runtime_profile=production`; `development` falls back to the first `@aztec/accounts/testing` account) |
| `TOPUP_AUTOCLAIM_SECRET_KEY_REF` | — | KMS / secret-manager reference for the claimer key |
| `TOPUP_AUTOCLAIM_SPONSORED_FPC_ADDRESS` | — | Use a sponsored FPC to pay the claim tx's fees (recommended — avoids needing Fee Juice on the claimer account) |

| `L1_RPC_URL` | If bridge | L1 RPC endpoint |
| `ADMIN_API_KEY` | No | For attestation token registration (admin endpoints) |

## Runtime Profiles

| Profile | Behavior |
|---------|----------|
| `development` | Permissive — allows plaintext secrets in config |
| `test` | Similar to development |
| `production` | **Strict** — rejects plaintext config secrets; requires `env`, `kms`, or `hsm` |

:::danger
Always use `runtime_profile: production` in production. It prevents accidental plaintext secret leaks.
:::

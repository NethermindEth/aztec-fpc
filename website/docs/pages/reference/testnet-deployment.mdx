# Testnet Deployment [Nethermind-operated public FPC]

Use these values to integrate against the **Nethermind-operated FPC on Aztec testnet** — no need to run your own attestation or top-up service. This is the deployment every other page links to in code examples.

:::info[When to use this]
- Learning the SDK without deploying your own stack
- Wallet development against a known, stable FPC
- Prototyping a cold-start flow end-to-end
- Running the [`examples/fpc-full-flow.ts`](https://github.com/NethermindEth/aztec-fpc/blob/main/examples/fpc-full-flow.ts) example

**Don't use this for production** — run your own FPC. See [Run an Operator](/how-to/run-operator).
:::

These are the addresses at time of writing. Always cross-check against the attestation service's live [`GET /.well-known/fpc.json`](https://aztec-fpc-testnet.staging-nethermind.xyz/.well-known/fpc.json) — that document is the source of truth. If the values above drift, the discovery document wins.
:::

## Quick verification

```bash
# Health
curl https://aztec-fpc-testnet.staging-nethermind.xyz/health

# Discovery document (authoritative for addresses + supported assets)
curl https://aztec-fpc-testnet.staging-nethermind.xyz/.well-known/fpc.json

# Current accepted assets + pricing
curl https://aztec-fpc-testnet.staging-nethermind.xyz/accepted-assets
```

## Notes on `humanUSDC`

The accepted token (`0x07348d12…`) is a **test token**, not a real USDC peg. It's deployed by the `configure-token` phase of the testnet stack and is claimable via the faucet contract. Do not confuse with mainnet USDC.

## Mainnet support

FPC is not yet deployed to Aztec Alpha Mainnet by Nethermind as a public service. Teams running FPC on mainnet should deploy their own stack — see [Run an Operator](/how-to/run-operator).

:::warning[Alpha Mainnet context]
Aztec Alpha Mainnet launched January 2026 (~3 months ago). Two critical security vulnerabilities have been disclosed since launch. This is early-mover territory — treat mainnet deployments accordingly, keep key management strict, and monitor the [Aztec security advisories](https://github.com/AztecProtocol/aztec-packages/security/advisories) for updates.
:::

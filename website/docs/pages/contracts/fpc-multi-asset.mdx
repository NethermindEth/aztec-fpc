# FPCMultiAsset Contract [The on-chain core that verifies quotes and pays gas]

The core fee payment contract. Verifies operator-signed quotes and pays transaction gas on behalf of users.

**Source:** `contracts/fpc/src/main.nr`

## Where is the "multi-asset" part?

**It's not an on-chain allowlist.** Readers landing on this page often expect storage to contain a list of accepted tokens. It doesn't — the contract holds a single immutable `Config` and that's it.

The "multi-asset" property comes from **quote binding + off-chain policy**:

| Layer | What enforces asset choice |
|---|---|
| On-chain (this contract) | `accepted_asset` is an argument to `fee_entrypoint` and is included in the signed quote preimage. Swapping the asset at call time invalidates the Schnorr signature, so the tx reverts. |
| Off-chain (attestation service) | The operator maintains the asset policy (exchange rates, fee spread) per asset in LMDB. The service refuses to sign quotes for unsupported assets. See [Add a Supported Asset](/how-to/add-supported-asset). |
| Protocol decision | [ADR-0001](/reference/asset-model-adr) — **one FPC deployment must support many assets** (no redeploying per token). |

In practice: adding a new accepted token is a `PUT /admin/asset-policies/:addr` call — no on-chain change, no redeploy. The same FPC contract instance serves USDC, ETH variants, your app's token, etc.

See [Security Model § No on-chain asset allowlist](/overview/security) for why this design is safe.

## Storage

```noir
struct Storage {
    config: PublicImmutable<Config>,
}
```

`Config` stores three immutable fields set at construction:

| Field | Type | Description |
|-------|------|-------------|
| `operator` | `AztecAddress` | Receives token payments |
| `operator_pubkey_x` | `Field` | Schnorr public key X coordinate |
| `operator_pubkey_y` | `Field` | Schnorr public key Y coordinate |

:::warning
These values are **immutable**. Operator key rotation requires deploying a new contract.
:::

## Constructor

```noir
#[public]
#[initializer]
fn constructor(
    operator: AztecAddress,
    operator_pubkey_x: Field,
    operator_pubkey_y: Field,
)
```

Stores the operator address and Schnorr public key coordinates. Called once at deployment.

## `fee_entrypoint`

Standard fee payment for users with existing L2 token balances.

```noir
#[private]
fn fee_entrypoint(
    accepted_asset: AztecAddress,    // Token contract address
    authwit_nonce: Field,            // Nonce for token transfer authwit
    fj_fee_amount: Field,            // Fee Juice amount (gas cost)
    aa_payment_amount: Field,        // Token amount (what user pays)
    valid_until: u64,                // Quote expiry timestamp
    quote_sig: [u8; 64],            // Operator's Schnorr signature
)
```

### Flow

::::steps

### Verify the quote

Calls `assert_valid_quote()`:
- Reconstructs the Poseidon2 hash with domain separator `0x465043`
- Verifies the Schnorr signature against stored operator pubkey
- Pushes the quote hash as a nullifier (prevents replay)
- Checks `valid_until >= block_timestamp` and `TTL <= 3600s`

### Enforce setup phase

Uses `set_as_fee_payer()` — can only execute during the transaction's setup phase.

### Transfer tokens

Enqueues a public call: transfer `aa_payment_amount` of `accepted_asset` from `msg_sender` to the operator using the provided `authwit_nonce`.

### Declare fee payer

The FPC contract pays the transaction's gas in Fee Juice.

::::

### Requirements

- Caller must have created an **authorization witness** (authwit) for the token transfer
- Quote must not be expired
- Quote must not have been used before
- Must execute during the tx setup phase

## `cold_start_entrypoint`

For new users who bridged tokens from L1 and have no L2 balance.

```noir
#[private]
fn cold_start_entrypoint(
    user: AztecAddress,              // User's L2 address
    accepted_asset: AztecAddress,    // Token contract address
    bridge: AztecAddress,            // Bridge contract address
    claim_amount: Field,             // Amount bridged from L1
    claim_secret: Field,             // Bridge claim secret
    claim_secret_hash: Field,        // Hash of claim secret
    message_leaf_index: Field,       // L1→L2 message index
    fj_fee_amount: Field,            // Fee Juice amount
    aa_payment_amount: Field,        // Token payment amount
    valid_until: u64,                // Quote expiry
    quote_sig: [u8; 64],            // Operator signature
)
```

### Flow

::::steps

### Guard: transaction root

Assert `msg_sender.is_none()` — this function **must** be the transaction's entrypoint (not called by another contract).

### Guard: claim covers fee

Assert `claim_amount >= aa_payment_amount` — user can't pay more than they're claiming.

### Verify cold-start quote

Uses `assert_valid_cold_start_quote()` with domain separator `0x46504373` and extended preimage (includes bridge, claim_amount, etc.).

### Declare fee payer

FPC pays gas in Fee Juice.

### Claim tokens from bridge

Enqueues `bridge.claim_public()` — mints tokens into FPC's public balance.

### Distribute tokens

- Transfer `claim_amount - aa_payment_amount` → **user**
- Transfer `aa_payment_amount` → **operator**

::::

### Requirements

- Must be **transaction root** (no parent caller)
- `claim_amount >= aa_payment_amount`
- Cold-start quote cannot be used with `fee_entrypoint` (different domain separator)

## Internal Helpers

### `assert_valid_quote`

```noir
fn assert_valid_quote(
    config: Config,
    accepted_asset: AztecAddress,
    fj_fee_amount: Field,
    aa_payment_amount: Field,
    valid_until: u64,
    quote_sig: [u8; 64],
    user: AztecAddress,
) -> Field  // returns quote_hash
```

Verifies a normal quote: hash, signature, nullifier, expiry, TTL.

### `assert_valid_cold_start_quote`

Same as above with extended preimage (bridge, claim_amount, claim_secret_hash, message_leaf_index) and domain separator `0x46504373`.

### `get_max_gas_cost`

Computes the maximum possible transaction fee using protocol gas settings.
